# 问题反馈通道
## 通道说明
为保障玩家所遇到的各类问题能够被官方与技术团队**及时接收、分类受理、跟进排查并正式修复**，请所有玩家根据自身实际问题类型，严格对应前往指定官方专属渠道规范提交反馈。

未按渠道分类、随意乱发反馈内容，将有可能导致问题延误处理、无法被对应负责人看到，进而影响问题排查与解决效率。

## 分类型官方反馈渠道
### 服务器相关问题
适用范围：服务器卡顿、频繁掉线、无法连接服务器、服务器大区异常、游戏内服务端故障、联机异常等服务端类相关问题。

反馈渠道：请进入官方 Discord 社区，在 `#报告问题 - report_problems` 专属板块内，完整描述问题现象、发生时间、触发条件等相关细节并提交。

**反馈格式**
```markdown
# Bug #{编号}
**问题描述：**
{问题描述}
**问题现象：**
{问题现象}
**问题触发条件：**
{问题触发条件}
```

如果你是要为服务器提出建议，请前往 `#建议反馈-suggest_feedback` 板块，而不是 `#报告问题 - report_problems`。

**反馈格式：**
```markdown
# Suggestion #{编号}
**建议描述：**
{建议描述}
**建议理由：**
{建议理由}
```
::: tip
编号请根据当前上下文获取，例如：上一位是#9，你则为#10。

suggest_feedback与report_problems的编号是独立的, 互不干扰。
:::
[https://discord.gg/k63hRWt3fF](https://discord.gg/k63hRWt3fF)

### 整合包相关问题
适用范围：游戏启动崩溃、模组报错闪退、模组版本冲突、整合包更新异常、启动器报错、资源加载失败等整合包与模组类问题。

反馈渠道：请前往 GitHub 平台，进入仓库 `EndlessPixel / EndlessPixel-Modpack`，在 **Issues** 区域按模板发起工单，详细描述问题日志与复现步骤。
[https://github.com/EndlessPixel/EndlessPixel-Modpack/issues](https://github.com/EndlessPixel/EndlessPixel-Modpack/issues)

### 官方网站相关问题
适用范围：官网无法访问、页面加载失败、网站功能异常、界面显示错乱、登录注册异常等官网相关故障。

反馈渠道：请前往 GitHub 平台，进入仓库 `EndlessPixel / server`，在 **Issues** 区域发起工单进行问题反馈。
[https://github.com/EndlessPixel/server/issues](https://github.com/EndlessPixel/server/issues)

## 反馈纪律须知
官方玩家社群、聊天群仅作为玩家日常闲聊、游玩交流、经验分享的非正式沟通场所，**不承接任何正式问题反馈与故障报修**。

严禁在任意玩家群内发布故障问题、求助报修、工单类反馈内容。凡在群聊内随意提交的问题反馈，一律判定为**无效反馈**，官方及技术团队不予记录、不予受理、不予跟进处理。

所有需要官方介入排查、修复、答复的问题，均须严格走上方指定官方通道提交。
