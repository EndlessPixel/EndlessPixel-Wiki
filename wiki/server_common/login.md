# 登录与注册

::: info
此文档中`<参数>`表示必须输入的参数，`[参数]`表示可选参数。
:::

### 🔐 /register <密码> <重复密码> /reg <密码> <重复密码>（注册账号）
这组命令是服务器为新用户提供的账号注册入口，完整指令/register和简写指令/reg功能完全一致，满足不同用户的操作习惯。使用时需在指令后依次输入自定义密码和重复密码，两次输入的密码必须完全一致才能通过系统校验。同时，服务器会对密码强度进行检测，要求密码需包含数字、字母、特殊字符的组合且长度达标，以此避免简单密码带来的账号安全风险。只要符合所有要求，用户就能成功创建专属账号，开启后续的服务器使用之旅。

### 🔐 /login <密码> /l <密码>（登录服务器）
这是已注册用户登录服务器的核心指令组合，/login为完整指令，/l为简写指令，二者均用于账号身份验证。用户每次退出服务器后再次进入，都需要输入对应指令并紧跟注册时设置的密码，系统会将输入的密码与数据库中存储的账号密码进行比对。验证通过后，用户即可解锁账号对应的全部权限和功能；若密码输入有误，系统会立即提示登录失败，多次输入错误还可能触发账号临时锁定机制，有效防范恶意登录行为，保障账号的安全性。

### 🔐 /changepassword <旧密码> <新密码> <重复新密码> /changepw <旧密码> <新密码> <重复新密码>（修改密码）
这组命令是服务器官方设定的密码修改工具，/changepassword是完整指令，/changepw是简写指令，二者规则和作用完全相同，专为需要更换账号密码的用户设计。使用时必须严格按照“旧密码+新密码+重复新密码”的顺序输入内容，系统会先验证旧密码的正确性，只有旧密码输入无误，才能进入新密码的设置环节。新密码需要满足服务器的安全规则，且两次输入的新密码必须完全一致，防止因输入失误导致密码设置失败，整个流程既保障了账号的所有权验证，又能提升密码的安全性。

### 🔐 /bindemail set <邮箱地址> /bdmail set <邮箱地址>（绑定邮箱）
这组命令用于为服务器账号绑定安全邮箱，/bindemail set是完整指令，/bdmail set是简写指令，二者均可实现邮箱绑定功能。绑定邮箱是账号安全的重要保障，不仅能用于密码找回，还能接收服务器的重要通知。使用时需在指令后输入有效的邮箱地址，系统会向该邮箱发送一封验证邮件，用户只有在规定时间内完成邮箱验证，才算真正完成绑定流程。需要注意的是，一个邮箱通常只能绑定一个服务器账号，避免出现账号管理混乱的情况。

### 🔐 /bindemail verify <验证码> /bdmail verify <验证码>（验证邮箱验证码）
这组命令是邮箱绑定流程的关键步骤，/bindemail verify为完整指令，/bdmail verify为简写指令，专门用于验证邮箱收到的验证码。用户在执行邮箱绑定指令后，会收到系统发送的包含验证码的邮件，此时输入对应指令并填写验证码，系统会快速校验验证码的正确性和时效性。若验证码准确无误，邮箱绑定将立即生效；若验证码输入错误或已过期，系统会提示验证失败，用户需要重新发送验证邮件并获取新的验证码，确保邮箱绑定流程的严谨性。

### 🔐 /resetpassword forget /repw forget（忘记密码重置）
这组命令是服务器为忘记密码的用户提供的找回入口，/resetpassword forget是完整指令，/repw forget是简写指令，二者功能一致。用户在忘记账号密码时，输入对应指令即可触发密码重置流程，系统会自动检测该账号是否已绑定安全邮箱。若邮箱已绑定，会立即向绑定邮箱发送验证码；若未绑定邮箱，系统会提示用户先完成邮箱绑定操作。整个命令的设计旨在帮助用户快速找回账号控制权，同时通过邮箱验证的方式，避免账号被他人恶意重置，保障账号的归属安全。

### 🔐 /bindemail re <验证码> <新密码> /bdmail re <验证码> <新密码>（使用验证码重置密码）
这组命令是密码重置流程的最后一步，/bindemail re是完整指令，/bdmail re是简写指令，用于通过验证码设置新密码。用户在执行忘记密码重置指令后，会收到邮箱验证码，此时输入对应指令，依次填写验证码和新设置的密码，系统会先校验验证码的有效性。验证通过后，会进一步检测新密码是否符合安全规则，确认无误后即可完成密码重置。新密码生效后，用户可以使用新密码登录服务器，整个流程既便捷又安全，有效解决了用户忘记密码的困扰。

::: warning
所有命令区分大小写，请正确输入。遇到问题可联系管理员帮助。
:::
