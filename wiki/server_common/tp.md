# 传送

::: info
文档中 `<参数>` 为必填参数，`[参数]` 为可选参数。所有指令**区分大小写**，输入时请确保准确。
:::

### 🚀 /rtp（随机传送）
随机传送到地图上的一个安全位置，适合用来探索新区域、找资源、远离危险区域，是日常探索最方便的传送方式之一。
输入指令后系统会自动寻找一个随机的安全地点，并把你传送过去。这个指令不需要指定目标玩家，完全随机，所以每次使用都会有不同的结果。适合想要冒险、发现新地方的玩家使用。

### 🚀 /tpa <玩家名>（请求传送到某玩家）
这是服务器里最常用的找人传送指令，作用是申请传送到指定玩家的位置。使用时在指令后面直接输入对方的游戏名，系统就会向他发送传送请求。对方可以选择接受或拒绝，只有对方同意后，你才会被传送到他当前所在的位置；如果对方拒绝或者长时间没回应，请求就会自动失效。平时找朋友、组队、一起玩都靠这个指令，输入时一定要把名字打对，打错会提示“未找到该玩家”。

### 🚀 /tpahere <玩家名>（请求某玩家传送到自己身边）
和/tpa正好相反，这个指令是邀请别人传送到你这里。输入指令加玩家名后，对方会收到你让他过来的传送请求。对方同意后，就会直接被传送到你的位置；拒绝则请求失效。平时找到好资源、打BOSS、集合活动、喊队友过来，用这个最方便。

### 🚀 /tpaignore <玩家名>（忽略某玩家的传送请求）
用来屏蔽某个玩家的所有传送请求，避免被频繁骚扰。输入玩家名后，这个玩家之后发给你的任何/tpa或/tpahere请求，你都不会收到提示，并且会直接失效。如果之后想恢复接收他的请求，可以用/tpatoggle指令解除。

### 🚀 /tpauto（自动接受他人传送请求）
开启自动接受所有传送请求。输入后，任何人发给你的/tpa和/tpahere都会自动同意，不用你再手动点接受。这个功能是对所有玩家生效的，适合信任的朋友之间使用，不建议随便开启，避免被陌生人乱传打扰。

### 🚀 /tpaccept（接受传送请求）
当你收到别人的传送请求时，输入这条指令就代表同意传送。如果对方是/tpa，他就会传到你这里；如果对方是/tpahere，你就会传到他那里。必须在请求还没超时的时候使用，超时后请求会自动失效。

### 🚀 /tpatoggle /tpatoggle <玩家名>（开启/关闭传送请求权限）
用来开关别人能不能给你发传送请求，有两种用法：
直接输/tpatoggle：全局开关，关闭后所有人都不能给你发传送请求。
输/tpatoggle <玩家名>：只对这一个玩家开关，屏蔽或允许他给你发请求。
每用一次就切换一次状态，非常适合精准管理谁能传你。

### 🚀 /tpdeny（拒绝传送请求）
和 /tpaccept 对应，用来拒绝别人的传送请求。输入后对方会收到被拒绝的提示，传送不会生效。正在打怪、建房子、不想被打扰的时候都可以用。

### 🚀 /back（传送回上一位置）
用来回到上一个位置。
直接输/back：你自己传送回刚才的位置，不小心掉坑、传错地方都能立刻救回。
这个指令只能回到最近一次记录的位置。

### 🚀 /tpcancel <玩家名>（取消向特定玩家的传送请求）
用来撤回你刚才发给别人的传送请求。输完玩家名就能取消还没被处理的/tpa或/tpahere。如果对方已经同意或拒绝了，就无法再取消，系统会提示“请求已失效，无法取消”。

::: warning
所有命令区分大小写，请正确输入。遇到问题可联系管理员帮助。
:::
